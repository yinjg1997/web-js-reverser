# (self.webpackChunkpc_web = self.webpackChunkpc_web || []).push()
这个是webpack多文件加载, 通常的做法是扣下来后将self改成window, 然后找`加载器`所在的`自执行函数``所在的文件` 然后全扣下来.

自执行函数所在的文件, 通常是 webpack 开头的,

一般在webpack前端页面中使用某个函数时,会通过`某个导出函数`拿到这个函数, 然后调用, 而不是直接通过`加载器函数`调用

# 在补 类webpack加载器 的环境时, 有两种做法
1. 全拿下来, 然后补环境
2. 只拿加密函数, 然后运行, 看缺什么函数补什么函数, 最后在补环境
